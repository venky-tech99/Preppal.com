Data Structures - Learn

What are Data Structures?

A Data Structure is a way of organizing and storing data so that it can be accessed and modified efficiently. In programming and computer science, data structures are used to manage large amounts of data efficiently for various operations such as searching, insertion, deletion, and updating.

Example: Imagine a bookshelf. The way books are arranged (alphabetically, by genre, by size) is similar to how data structures organize data.

Types of Data Structures

  • Primitive Data Structures: int, char, float, boolean, etc.
  • Non-Primitive Data Structures: Arrays, Linked Lists, Stacks, Queues, Trees, Graphs, Hash Tables.

Array

An array is a collection of elements stored at contiguous memory locations.

[10] [20] [30] [40] [50] Index → 0 1 2 3 4 
// Example: Array in C
int arr[5] = {10, 20, 30, 40, 50};
printf("%d", arr[2]); // Output: 30

Linked List

A Linked List is a linear data structure where elements are stored in nodes, and each node contains a data field and a reference (link) to the next node.

[Data|Next] → [Data|Next] → [Data|NULL] 
// Example: Singly Linked List Node in C
struct Node {
    int data;
    struct Node* next;
};

Stack

A Stack follows the LIFO (Last In First Out) principle.

TOP → [40] [30] [20] [10]
Example: Undo operation in text editors uses a stack.

Queue

A Queue follows the FIFO (First In First Out) principle.

Front → [10] [20] [30] [40] ← Rear
Example: People standing in a line for tickets.

Tree

A Tree is a hierarchical data structure with nodes connected by edges.

Root ├── Child1 └── Child2

Graph

A Graph is a collection of nodes (vertices) connected by edges.

(A) --- (B) | | (C) --- (D)

Hash Table

A Hash Table stores key-value pairs and uses a hash function to compute the index for storing data.

Applications of Data Structures

  • Database indexing (B-Trees, Hash Tables)
  • Network routing (Graphs)
  • Operating system scheduling (Queues)
  • Compiler syntax checking (Stacks)
