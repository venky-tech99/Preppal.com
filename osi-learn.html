OSI Model — Overview

The OSI (Open Systems Interconnection) model is a conceptual framework of seven layers that explains how networks communicate. For interviews memorize responsibilities of each layer and typical examples (protocols & devices).

7

Application Layer

User-facing services (HTTP, SMTP, FTP). Handles application data.
6

Presentation Layer

Formats/encodes data (encryption, compression).
5

Session Layer

Manages sessions/connection dialogs.
4

Transport Layer

End-to-end communication (TCP, UDP). Handles reliability, flow control.
3

Network Layer

Routing between networks (IP). Decides path.
2

Data Link Layer

Frames, MAC addressing, switches. Detects/corrects physical errors.
1

Physical Layer

Wires, signals, physical media, NIC speed.

Placement-focused Example

When you open a webpage (http): Application layer (browser) creates request → Transport (TCP) ensures ordered delivery → Network (IP) routes packets → Data Link/Physical transport physically. Mention ports (HTTP = 80, HTTPS = 443) during interviews.

Common pitfalls to explain

  • Don’t confuse Presentation (formatting/encryption) with Application (service-level)
  • Transport (TCP) provides reliability; UDP does not
Interview Qs
  1. Explain TCP vs UDP and when to use each.
  2. Which layer handles routing and which handles switching?
  3. Why encryption is not at the transport layer?
