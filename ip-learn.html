IP Addressing & Subnetting

IPv4 Essentials

IPv4 addresses are 32-bit numbers written as dotted-decimal (e.g., 192.168.1.0). Subnet masks split network and host portions (e.g., /24 = 255.255.255.0).

How subnetting works

Given network 192.168.10.0/24, dividing into /26 gives subnets with 64 addresses (62 usable). CIDR notation /n indicates n bits for network prefix.

Example:
/24 → hosts per subnet = 2^(32-24) = 256 (254 usable)
/26 → hosts = 2^(32-26) = 64 (62 usable)

Common commands

Linux:
ip addr show
ip route show
ping 8.8.8.8
ifconfig (older)
Quick calculation

Question: How many hosts in /27? Answer: 2^(32-27) = 32 addresses (30 usable).

IPv6 basics

IPv6 uses 128-bit addresses; colon-hex notation (e.g., 2001:0db8::1). No NAT required; much larger address space.
