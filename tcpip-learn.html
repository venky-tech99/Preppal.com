TCP/IP Model — Placement Overview

TCP/IP — Core idea

The TCP/IP model is used practically for internet communications. It has 4 layers and maps roughly to OSI. Interviewers expect exact layer roles, common protocols, and mapping to OSI.

Application
Transport
Internet
Network Access
Application: HTTP, SMTP, DNS — user facing protocols
Transport: TCP (reliable), UDP (fast)
Internet: IP routing, ICMP
Network Access: Ethernet, ARP, physical link

Compare to OSI

TCP/IP collapses Presentation/Session into Application; Network Access = Data Link + Physical. For interviews show mapping table below.

Example: HTTP Request Flow

Browser (Application) → TCP (handshake, reliability) → IP (routing) → Ethernet (frame on link). A common interview question: explain the three-way TCP handshake (SYN, SYN-ACK, ACK).

Interview Qs
